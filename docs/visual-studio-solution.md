# Visual Studio Solution Layout

This repository generates a single canonical Visual Studio solution:

- `Builds/universal_agent_manager.sln`

Open that solution when working on the app in Visual Studio. The main executable is the only project you normally need to run.

## What You Should Open

- `universal_agent_manager` is the startup project and desktop app.
- Use that project for day-to-day development, debugging, and running the app.

## Solution Folders

The solution is grouped to keep the project list readable:

| Folder | What it contains |
| --- | --- |
| `App` | The main executable project: `universal_agent_manager` |
| `ThirdParty/SDL2` | Main SDL2 dependency target (`SDL2-static`) |
| `ThirdParty/SDL2/Internal` | SDL2 helper/internal targets (`SDL2main`, `sdl_headers_copy`, optional `uninstall`) |
| `_CMake` | Utility targets generated by CMake and Visual Studio |

## Real Projects

These are the projects that represent application code:

- `universal_agent_manager`

## Generated Dependency Targets

These come from SDL2 and should be treated as dependency internals, not app features:

- `SDL2-static`
- `SDL2main`
- `sdl_headers_copy`
- `uninstall` if your SDL2 revision exposes it

## Generated Utility Targets

These are created by CMake or Visual Studio to support builds and testing:

- `ALL_BUILD`
- `ZERO_CHECK`
- `RUN_TESTS` when tests are enabled
- `Continuous`
- `Experimental`
- `Nightly`
- `NightlyMemoryCheck`

## Folder And File Noise

You may also see additional `.sln` files under `Builds/_deps/`. Those belong to fetched dependencies and are not the main UAM solution.

If the workspace starts to feel crowded, focus on `Builds/universal_agent_manager.sln` and ignore the generated dependency solutions.
